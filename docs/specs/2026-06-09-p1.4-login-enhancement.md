> 2026-06-09 无人值守。Claude × Codex medium PK 收敛（机制层完全一致）。本文冻结决策供实现 +
> 用户 review。**⚠️ 标「待确认」的是 Codex 裁决「命中升级条件」的参数/形式选择**——已用 Codex

## 1. 机制层（双方收敛，业界标准，已自动采纳）

### 1.1 refresh token = opaque + HMAC 落库可撤销

- token 明文 `rt_<jti>.<secret>`：`jti` UUID、`secret` 256-bit 随机（`secrets.token_urlsafe`）。
- DB **只存** `token_hash = HMAC-SHA256(pepper, secret)`（不存明文）。高熵 token 用 HMAC+pepper
  而非 Argon2（Argon2 适合低熵密码；refresh 要高频等值校验）。pepper 独立于 `auth_jwt_secret`（泄露隔离）。
- **不用无状态 JWT refresh**（无法撤销，与「落库可撤销」冲突；P0 spec ADR-C 已定）。

### 1.2 表 `auth_refresh_tokens`（IdMixin + TimestampMixin + 全列中文 comment）

| 列 | 说明 |
|---|---|
| `jti` UUID uq | 当前 token 标识 |
| `family_id` UUID | 轮换链（一次登录 = 一个 family） |
| `user_id` BIGINT FK users.id CASCADE | 所属用户 |
| `token_hash` CHAR(64) uq | HMAC-SHA256 hex |
| `rotated_to_jti` UUID null | 轮换后继 |
| `revoked_at` timestamptz null | 撤销时间 |
| `revoked_reason` VARCHAR(32) null | rotated/logout/reuse_detected/concurrency_limit/expired_cleanup |
| `issued_at` / `expires_at` timestamptz | 签发/过期 |
| `last_used_at` timestamptz null | 最后轮换时间 |

索引：`uq jti`、`uq token_hash`、`ix (user_id, family_id)`、`ix (user_id, revoked_at, expires_at)`、`ix expires_at`。
device 信息（ua/ip）**只审计不强绑定校验**（后台 IP/UA 常变，强绑定误杀）—— P1.4 暂不落 device 列。

### 1.3 rotation + reuse detection（RFC 9700）

- `/login` 成功 → 新 `family_id` + 初始 token row + access token。
- `/refresh` → 按 `jti` 查 row（`SELECT FOR UPDATE` 防并发竞态）→ HMAC 比对 secret →
  - 有效：插新 row，旧 row `rotated_to_jti=new` + `revoked_at` + `revoked_reason='rotated'`，返回新 access+refresh。
  - **已 rotated 的 token 再用（reuse）→ 撤销整个 family** + `reuse_detected`（token theft 信号）。
  - 不存在/hash 不符/过期/已撤销 → 统一 `auth.REFRESH_TOKEN_INVALID`（不暴露细节）。
- 并发上限：login 时该 user 活跃 family 数 > 上限 → 撤销最旧 family（`concurrency_limit`）。
- 过期清理：惰性（查询过滤 `expires_at`）+ CLI `auth cleanup-refresh-tokens`（不引 scheduler，P4 才做定时）。

### 1.4 验证码（纵深，非主防线）

- 自建**算术文本**验证码（如 "3 + 5 = ?"），**不引图形库**（规避新依赖 + P0 spec「避免 Pillow」倾向）。
- Redis `auth:captcha:{id}` = answer_hash + attempts，TTL 120s；成功校验立即 DEL（一次性）。
- 不每次登录都要 —— 失败 N 次后才要求（Q14：否则验证码=安全表演）。

### 1.5 登录限流（Q14 联动，组合维度）

- Redis 计数维度：`auth:fail:user:{username}`（防分布式 IP）+ `auth:fail:ip:{ip}`（防单源撞库）+ combo。
- 阈值：失败 ≥3 → 要求 captcha；≥5 → 账号软锁 10min；IP ≥30/窗口 → 429 + `Retry-After`。
- **Redis 不可用 fail-closed**（认证防护 ≠ 幂等 fail-open）：降级为「要求验证码」，不完全放行。

## 2. 端点契约

- `POST /api/v1/auth/login`：`LoginResponse` 增 `refresh_token` + `refresh_expires_in`（向后兼容新增）。
  失败 ≥ 阈值时返回 `auth.CAPTCHA_REQUIRED`（前端据此拉 captcha）。
- `POST /api/v1/auth/refresh`：`{refresh_token}` → 同 login 成功响应（强制 rotation）。
- `POST /api/v1/auth/logout`：`{refresh_token, all_devices?}` → 撤销 family（或该 user 全部）→ 204。
- `GET /api/v1/auth/captcha`：→ `{captcha_id, question, expires_in}`（算术文本）。
- 公开路径（免 token）：login / refresh / logout / captcha。

错误码：`auth.REFRESH_TOKEN_INVALID` · `auth.REFRESH_TOKEN_REUSED` · `auth.CAPTCHA_REQUIRED` ·
`auth.CAPTCHA_INVALID` · `auth.LOGIN_RATE_LIMITED`。

## 3. ⚠️ 待用户最终拍板（Codex 裁决「命中升级条件」，已用推荐默认实现）

| 决策 | 推荐默认（已实现，settings 可配） | 备选 |
|---|---|---|
| refresh token 存放 | **body**（前端内存持有） | httpOnly cookie（须配 CSRF/SameSite，留 P6） |
| 验证码形式 | **算术文本**（不引依赖） | 图形 base64（引 Pillow/captcha 新依赖） |
| 并发 family 上限 | **5** | 3 / 10 |
| refresh TTL | idle 7d / absolute 30d | 后台可更短 |
| access TTL | **暂保 2h** | 收到 30min（P0 spec 倾向，本阶段未改，待确认） |
| 账号软锁对外错误码 | captcha_required 显式 + 账号锁**统一 401** + IP 级 429 | 显式 `auth.LOGIN_LOCKED`（前端友好但泄露账号存在性） |
| audit EventType 扩展 | 加 `login_success` / `refresh_reused` / `logout` | 不扩展（塞 metadata，不建议） |

## 4. 分阶段实现（每 slice verify + commit）

1. **refresh DB slice**：settings + security 原语 + model + migration 0010 + repository（rotation/reuse/concurrency）+ 单测。
2. **token API slice**：login 加 refresh + `/refresh` + `/logout` + LoginResponse 扩展 + 端到端。
3. **captcha slice**：Redis captcha store + `/captcha` + 一次性消费测试。
4. **rate limit slice**：Redis login guard（组合维度 + fail-closed）+ 阈值测试。
5. **hardening**：审计 EventType 扩展 + cleanup CLI + 文档。

## 5. 风险（Codex 标，实现须覆盖）

- refresh 并发竞态 → `SELECT FOR UPDATE`。
- reuse 误伤（多 tab/重试）→ 前端单飞 refresh + 服务端明确重登信号。
- Redis 降级 → 限流 fail-closed（要求验证码）。
- 账号锁定被滥用 → 组合维度 + 短锁 + 超管应急。
- 审计泄密 → refresh_token / captcha answer / Authorization 不入日志（audit deny-list 已覆盖）。
