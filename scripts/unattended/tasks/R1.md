## 真实 PermissionProvider（核心，O2 归一）

新文件 `src/admin_platform/domains/role/provider.py`（domains 层，**可 import dept/user/role repository**；authz 抽象不能 import domains，但 domains 实现可以）。`class DbPermissionProvider(PermissionProvider)`：

- `get_is_super_admin(user_id)`：查 `users.is_super_admin`。
- `get_effective_data_scope(user_id) -> DataScope`（**O2 归一**）：
  - 查 user 的 roles（各 `data_scope`）。
  - 任一 `ALL` → `return DataScope(ScopeType.ALL, user_id)`。
  - 否则归一：`visible_dept_ids` = 各角色部门范围**并集** —— `SELF_DEPT`→{user.dept_id}；`SELF_DEPT_AND_BELOW`→`dept_repo.list_descendant_dept_ids(user.dept_id)`；`CUSTOM_DEPT`→`list_custom_dept_ids_for_role(role_id)`。`include_self` = 任一角色为 `SELF`。返回 `DataScope(ScopeType.CUSTOM_DEPT, user_id, visible_dept_ids=并集, include_self=...)`（scope_type 设非 ALL 即可，apply 用 visible + include_self）。
  - user 无 dept_id 时 `SELF_DEPT`/`AND_BELOW` 贡献空集（安全 deny，不报错）。
  - 无角色 → `DataScope(ScopeType.CUSTOM_DEPT, user_id, visible_dept_ids=frozenset(), include_self=False)`（apply → false deny）。
- `get_user_permissions(user_id)`：**R1 先返回 `frozenset()`**（权限标识来自 `menu.perms`，ME1 实现；R1 不做 permissions）。
- `invalidate_*`：no-op（P1 不缓存）。
