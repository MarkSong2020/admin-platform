## 背景

A1 已建 `src/admin_platform/authz/scope.py`（`ScopeType` + `DataScope`）。本任务加把 `DataScope` 翻译成 SQLAlchemy `where` 的**纯函数** —— 这是数据权限的落地层（spec §4.2：声明式负责"谁要权限"，本 helper 负责"SQL 怎么加条件"）。

## 范围

1. `src/admin_platform/authz/data_scope.py`：
   - `def apply_data_scope(stmt: Select, scope: DataScope, *, dept_col, owner_col) -> Select`
   - 按 `scope.scope_type` 追加 where（用 `stmt.where(...)`，返回新 stmt）：
     - `ALL` → 不加条件，原样返回
     - `SELF` → `owner_col == scope.user_id`
     - `SELF_DEPT` → `dept_col == scope.dept_id`
     - `SELF_DEPT_AND_BELOW` / `CUSTOM_DEPT` → `dept_col.in_(scope.visible_dept_ids)`
   - `dept_col` / `owner_col` 是调用方（repository）传入的 SQLAlchemy 列。类型标注用 `ColumnElement` 或 `Any`（按 pyright 通过为准）。
   - 中文 docstring 说明每个分支。

2. `tests/unit/test_apply_data_scope.py`：用一个最小 SQLAlchemy 模型（或 `literal_column`）断言每个 ScopeType 生成的 where 正确。**安全关键，负向断言要全**：
   - `SELF` 范围下，编译出的 SQL 含 `owner = :user_id`，**不含**跨 user 条件
   - `ALL` 不加任何 where
   - 部门范围用 `IN (visible_dept_ids)`
   - 可用 `str(stmt.compile(compile_kwargs={"literal_binds": True}))` 断言 where 子句
