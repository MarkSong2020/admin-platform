## models.py（depts 表，邻接表）

继承 `Base, IdMixin, TimestampMixin`。字段（**全列中文 comment**）：
- `parent_id: Mapped[int | None]` BIGINT NULL, `ForeignKey("depts.id", ondelete="RESTRICT")`, comment="父部门ID(NULL=根)"
- `name: String(64)` NOT NULL, comment="部门名称"
- `code: String(64)` NOT NULL, comment="部门编码"
- `sort_order: int` NOT NULL default 0, comment="显示顺序"
- `status: String(16)` NOT NULL default "active", comment="状态(active/disabled)"
- `leader / phone / email`: 可空, 中文 comment

`__table_args__`：
- `UniqueConstraint("code", name="uq_depts_code")` + `register_unique_constraint("uq_depts_code", "dept.CODE_DUPLICATE", "Dept code already exists")`
- `CheckConstraint("parent_id IS NULL OR parent_id <> id", name="ck_depts_not_self_parent")`
- `Index("ix_depts_parent_sort", "parent_id", "sort_order", "id")`

## repository.py

- 标准 CRUD（create/get/list/update/delete；select 风格；count 用 `select(func.count()).select_from(Dept)`）。
- `async def list_descendant_dept_ids(self, dept_id: int) -> frozenset[int]`：recursive CTE（base = `select(Dept.id).where(Dept.id == dept_id)`；`union_all` child `parent_id == tree.c.id`），返回**含自身**的子孙 id 集合。
- `async def list_ancestor_dept_ids(self, dept_id: int) -> list[int]`（面包屑，递归向上，返回祖先 ID 链）。

## service.py（业务规则，抛 AppError、错误码 `dept.*`）

- CRUD 用例；create/update 校验 code 唯一（依赖 register 的 `dept.CODE_DUPLICATE`，409）。
- **移动防环**：update `parent_id` 时，new_parent_id 不能是自身、不能在自身子孙集合内（用 `list_descendant_dept_ids` 校验）；违反抛 `AppError(dept.CYCLE, 409)`。
- **删除 RESTRICT**：有子部门时禁删（抛 `AppError(dept.HAS_CHILDREN, 409)`）。
